### Minor: 2

#### MINOR-1: `verify_deploy.py` misses `balance_hkd` / `balance_btc` in REQUIRED_COLUMNS

**File:** `backend/verify_deploy.py`  
**Section:** `REQUIRED_COLUMNS` dict, `"wallets"` list

The verification script checks for `balance_native`, `balance_usd`, and whale score columns, but does **not** include `balance_hkd` and `balance_btc` — columns added in migration 012 that the frontend (`Wallets.jsx:26-28`, `Dashboard.jsx:51-53`) and the `list_wallets` endpoint (`main.py:857-858`) both read.

If migration 012 is not applied, the `/api/wallets` and `/api/dashboard` endpoints will raise `UndefinedColumnError` at runtime, but `verify_deploy.py` will still report a passing schema check.

**Fix:** Add `"balance_hkd"` and `"balance_btc"` to the `REQUIRED_COLUMNS["wallets"]` list.

#### MINOR-2: `verify_deploy.py` lists `alerts.cooldown_seconds` which doesn't exist in any migration

**File:** `backend/verify_deploy.py`  
**Section:** `REQUIRED_COLUMNS["alerts"]` list

The column `cooldown_seconds` is listed as required in the verification script, but **no migration** creates this column. The alert cooldown logic in `alert_evaluator.py` uses an in-memory `_cooldown_cache` dict, not a DB column. This means the schema check will always fail on this column, but it's marked as `"critical"` severity, which would block deployment even though the actual alert cooldown mechanism works correctly from memory.

**Fix:** Remove `"cooldown_seconds"` from `REQUIRED_COLUMNS["alerts"]`. Add `"last_fired_at"` (which IS in migration 009 and IS correct).

### Observations (non-blocking)

#### OBS-1: Signal dedup relies on `(wallet_id, token_symbol, action, amount_usd)` unique constraint

The `signal_generator.py:247` INSERT uses `ON CONFLICT (wallet_id, token_symbol, action, amount_usd) DO NOTHING` but the constraint is created in migration 010 as `uq_copy_trade_signals_dedup`. If migration 010 was not applied (e.g., only up to 009 ran), duplicates would silently insert. The migration includes a dedup cleanup step before adding the constraint, which is good.

**Recommendation:** Add `uq_copy_trade_signals_dedup` to `REQUIRED_UNIQUE_CONSTRAINTS` in `verify_deploy.py`.

#### OBS-2: Price cache uses two different CoinGecko endpoints

The monitor's `_price_cache` (`monitor.py:648`) fetches from `simple/price` for `ethereum` with `vs_currencies=usd,hkd,btc`, then a second request for `solana,bitcoin`. The dashboard price cache (`main.py:89`) also fetches from the same endpoint. Under CoinGecko rate limits (10-30 calls/min free tier), this is 3 requests every 60 seconds. If both the monitor and dashboard cache expire in the same cycle, it could burst to 4+ requests.

**Recommendation:** Consider consolidating both price fetches into a single module-level cache with a single CoinGecko call.

#### OBS-3: `poll_wallets` migration cleanup references wrong module

Migration 010 line 1: `-- Fix: Add unique constraint on copy_trade_signals ... The INSERT in poll_wallets.py line 151 uses ON CONFLICT DO NOTHING` — the module name `poll_wallets.py` doesn't exist; the actual code is in `monitor.py`. This is just a comment but could mislead future developers.

**Recommendation:** Update the comment to reference `monitor.py`.
