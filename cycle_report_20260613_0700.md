## Bug Fixed

### Bug: `TypeError: '>' not supported between instances of 'NoneType' and 'int'` in `_fetch_dashboard_prices()`
**Severity**: Critical (test failure, would crash production on first dashboard load)
**Root Cause (Pitfall #5)**: The `_dashboard_price_cache` dict was initialized with `"timestamp": None` (sentinel for "never updated"), and `_price_cache` in monitor.py had the same pattern. The fast-path check used:
```python
if cache.get("timestamp", 0) > 0 and now - cache["timestamp"] < 120:
```
`.get("timestamp", 0)` returns `None` (not `0`) because the key **exists** with value `None` — the default only applies when the key is **missing**. Then `None > 0` raises `TypeError`.

**Fix**: Replaced with explicit None-check pattern:
```python
_ts = cache.get("timestamp")
if _ts is not None and now - _ts < 120:
    return cache
```
